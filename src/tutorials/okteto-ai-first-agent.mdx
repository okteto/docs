This tutorial walks you through launching an AI agent from the Okteto Dashboard. You will give the agent a task, watch it build and test code in a live environment, review the results, and create a pull request.

## What you will learn

- How to launch an AI agent from the Okteto Dashboard
- How to write effective prompts that produce good results
- How to review agent output in the embedded editor and preview environment
- How to create a pull request from agent-generated changes

## Prerequisites

Before you start, make sure you have:

- Access to the Okteto Dashboard
- Your administrator has [enabled Okteto AI](/docs/okteto-ai/okteto-ai-admin-config) for your organization
- The **Agents** tab is visible in your dashboard
- A repository you want to work with (or you can start a new project)

:::tip
Don't see the **Agents** tab? Contact your administrator to request access.
:::

## Step 1: Open the Agents tab

Navigate to the **Agents** tab in your Okteto Dashboard. This is where you launch, monitor, and manage AI agents.

## Step 2: Choose a repository

Select **Work with an existing repository** and paste your repository URL. For this tutorial, you can use the Movies sample app:

```
https://github.com/okteto/getting-started
```

The agent clones and analyzes your codebase before starting work.

:::info
You can also select **New Project** to have the agent build something from scratch.
:::

## Step 3: Write your prompt

Write a clear, specific prompt that describes what you want. Include what to build, where it should go, and any constraints.

For this tutorial, enter the following prompt:

```
Add a health check endpoint at /api/health that returns a JSON
response with the server status, current timestamp, and uptime
in seconds. Add a unit test for the new endpoint.
```

**What makes this prompt effective:**

| Element | Example from the prompt |
|---------|------------------------|
| **What** to build | A health check endpoint |
| **Where** it goes | `/api/health` |
| **How** it should behave | Returns JSON with status, timestamp, uptime |
| **Verification** | Includes a unit test |

:::warning
Vague prompts like "add a health check" produce inconsistent results. Always specify the endpoint path, response format, and expected behavior.
:::

## Step 4: Launch the agent

1. Press **Enter** or click **Launch Agent**
2. Watch the real-time logs as the agent sets up the environment, reads the codebase, and starts coding
3. Click **Open Editor** to see a Visual Studio Code instance where you can monitor all file changes

The agent follows this workflow:

1. Deploys the application to an Okteto environment
2. Reads the codebase to understand the project structure
3. Creates the health check endpoint
4. Writes unit tests
5. Runs the tests to verify the implementation
6. Reports the results

## Step 5: Review the results

Once the agent completes its work:

1. **Check the preview**: Click the generated endpoint URL to see your application running. Navigate to `/api/health` to verify the health check response
2. **Review the code**: Use the embedded editor to inspect all file changes. Verify the endpoint logic, error handling, and test coverage
3. **Check test results**: Review the test output in the agent logs to confirm all tests pass

## Step 6: Create a pull request

When you're satisfied with the results, ask the agent to create a pull request:

```
Create a pull request with these changes
```

The agent commits the changes and opens a PR in your repository. Review the PR, request changes if needed, and merge when ready.

## Effective prompts

The quality of agent output depends on the quality of your prompt. Here are patterns that work well:

**Add an API endpoint:**
```
Add a REST endpoint at /api/users/:id/avatar that accepts image
uploads (JPEG/PNG only, max 5MB), resizes to 200x200, stores in
S3, and returns the CDN URL. Write tests for success and error cases.
```

**Fix a bug:**
```
The /api/orders endpoint returns a 500 error when the cart is empty.
Investigate the error, implement a fix that returns a 400 with a
descriptive error message, and add a regression test.
```

**Add tests:**
```
Write comprehensive unit tests for the OrderService class in
src/services/order.ts with at least 80% coverage. Include tests
for edge cases like empty orders and invalid quantities.
```

Include these details when relevant:
- **What**: The specific feature, fix, or task
- **Where**: File paths, endpoints, or service names
- **How**: Technical requirements and constraints
- **Why**: Business logic or user needs that inform implementation decisions

## Running agents in parallel

Multiple agents can work simultaneously on different tasks. Launch separate agents to:
- Add a feature in one agent while another writes tests
- Fix a bug in the API while another optimizes database queries
- Generate documentation while another refactors code

Each agent gets its own isolated environment, so they don't interfere with each other.

## Summary

You launched an AI agent from the Okteto Dashboard, gave it a well-structured prompt, reviewed the code and preview environment, and created a pull request. Agents work best when you provide clear requirements, verify the output in the preview environment, and iterate with specific feedback.
