The file `chart/values.yaml` has a password hardcoded. This is not a good security practice :-)

Luckily, [Okteto Variables](admin/dashboard.mdx/#variables) can be used to inject sensitive information at runtime in your deploy commands.

Create an Okteto Variable called `MONGODB_PASSWORD` from the [Okteto Admin Dashboard](admin/dashboard.mdx/#variables), and use it this way in your `okteto.yml` file to configure the MongoDB password:

```yaml
deploy:
  - helm upgrade --install movies chart
    --set api.image=${OKTETO_BUILD_API_IMAGE}
    --set frontend.image=${OKTETO_BUILD_FRONTEND_IMAGE}
    --set mongodb.password=${MONGODB_PASSWORD}
```
