You can use Personal Access Tokens instead of OAuth to authenticate with Okteto.

Personal Access Tokens are useful when scripting, or when configuring [Preview Environments](preview/overview.mdx).

### Creating a Personal Access Token

1. Sign in to your Okteto account.
1. Click on the **Settings** icon on the navigation bar at the left.
1. Click on the **New Token** button.
<p align="center"><Image src="/docs/core/images/tokens-new-sidebar.png" alt="new token" width="850" /></p>
1. Give your token a descriptive name, select the expiration, and click the **Generate** button.
<p align="center"><Image src="/docs/core/images/tokens-name.png" alt="token name" width="630" /></p>
1. Copy the token to your clipboard. For security reasons, after you navigate off the page, you will not see the token again.
<p align="center"><Image src="/docs/core/images/tokens-copy.png" alt="copy token to clipboard" width="630" /></p>

> Warning: Treat your tokens like passwords and keep them secret. Always use tokens as environment variables instead of hardcoding them into your programs.

### Token Expiration

When you create a token, you can select the expiration, the default expiration is 180 days. Once a token has expired, it can no longer be used to authenticate or for API requests. It is not possible to restore an expired token, you will need to create a new token upon expiration.

A banner will be displayed a week before your token expires to remind you of this. We recommend that you take the appropriate measures once you see this banner in order to prevent any potential disruptions.
<p align="center"><Image src="/docs/core/images/tokens-expiration-banner-sidebar.png" alt="tokens expiration banner" width="850" /></p>

The state and expiration date of every token will be displayed in the UI.
<p align="center"><Image src="/docs/core/images/tokens-expiration-list.png" alt="tokens expiration banner" width="850" /></p>

## Using a Token on the Command Line

Once you have a token, you can use it to authenticate with the [Okteto CLI](reference/okteto-cli.mdx) instead of using your browser, as shown below:

```console
$ okteto context use https://okteto.example.com --token $YOUR_TOKEN
```

Personal Access Tokens can also be used when setting the `OKTETO_TOKEN` environment variable.

```console
$ export OKTETO_TOKEN=xxxxxxx
$ okteto namespace create test-cindylopez
```

### Revoking a Personal Access Token

1. Sign in to your Okteto account.
1. Click on the **settings** icon on the left tab.
1. Click on the **Delete** button.
<p align="center"><Image src="/docs/core/images/tokens-delete-sidebar.png" alt="delete a token" width="850" /></p>
1. Click on the **Delete Token** button to confirm that you want to delete your token.
<p align="center"><Image src="/docs/core/images/tokens-delete-confirm.png" alt="confirm token deletion" width="478" /></p>

Once deleted, the token is automatically revoked, and it can't be recovered.
