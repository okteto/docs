The Okteto CLI is ourÂ [open-source](https://github.com/okteto/okteto)Â tool that let's you develop your applications on Okteto.

If you haven't done so yet, install and configure the Okteto CLI following [this guide](get-started/install-okteto-cli.mdx).

The Okteto CLI requires one of the following manifests to know how to deploy your application:

1. [Okteto manifest](core/okteto-manifest.mdx): if there is an `okteto.yml` or `.okteto/okteto.yml` file in your folder, Okteto will use this file to deploy your application
1. [Docker Compose](reference/docker-compose.mdx): if there is an `okteto-compose.yaml`, `docker-compose.yaml` or `compose.yaml` file, Okteto will use this file to deploy your application

:::note
You can use the `-f` flag on any Okteto CLI command to explicitly define the location of an Okteto Manifest.
:::

If you do not yet have an Okteto Manifest, we recommend to follow this tutorial to [create your first Okteto Manifest](get-started/deploy-your-app/index.mdx).

If you have a Docker Compose file, the Okteto CLI can directly deploy your application without generating an Okteto Manifest.
However, you can still choose to create an Okteto Manifest [referring to your Docker Compose file](https://www.okteto.com/docs/reference/okteto-manifest.mdx#deploy-with-compose) if you want more control over the configuration of your Development Environment.
