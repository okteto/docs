This guide explains how to onboard your own applications to Okteto. This guide includes the following steps:

- Configure Okteto to deploy your application
- Expose your application using Okteto Endpoints
- Build your container images with Okteto Build
- Configure an Okteto Variable for a database password

> If you have a Docker Compose file, you can use it with Okteto with no changes. Follow [this guide](guides/tutorials/compose-getting-started.mdx) for more details.

## Requirements

- You have access to an Okteto instance. [Sign up for our free 30 days trial](https://www.okteto.com/free-trial/) to start the process
- You have [installed the Okteto CLI](get-started/install-okteto-cli.mdx/#installing-the-okteto-cli) and [configured the Okteto CLI](get-started/install-okteto-cli.mdx/#configuring-the-okteto-cli) with your Okteto instance
- You have completed our [getting started guide](get-started/deploy-first-env.mdx) and understand the basics of the Okteto development experience

## Configure Okteto to deploy your application

We will be using our Movies App as a sample, but removing all the Okteto configuration first.

Start by forking the Movies App repo:

```console
$ git clone https://github.com/okteto/movies-with-helm
$ cd movies-with-helm
```

and checkout the `onboard-your-app` branch:
```console
$ git checkout onboard-your-app
```

The Movies App comes with a Helm chart to deploy the application on Kubernetes.
You can use any other tool to deploy your application with Okteto. The process to do that would be quite similar to the process followed in this guide.

To instruct Okteto to deploy this application, create an `okteto.yml` file at the root of the repo with the following content:

```yaml
deploy:
  remote: true
  commands:
    - name: Helm Install
      command: helm upgrade --install movies chart
```

Now you are ready to deploy your app with Okteto running `okteto deploy`:

```console
$ okteto deploy
```

## Expose your application using Okteto Endpoints

Okteto can be configured to automatically create endpoints for your application.
In order to do that, you need to add the following annotation to your [Ingress](https://github.com/okteto/movies-with-helm/blob/onboard-your-app/chart/templates/ingress.yaml) resource:

```yaml
annotations:
  dev.okteto.com/generate-host: "true"
```

This instructs Okteto to create an endpoint in the Okteto wildcard domain.
For example, if your development environment name is `movies` in the `cindylopez` namespace, it will be accessible at https://movies-cindylopez.okteto.example.com if your Okteto domain is okteto.example.com.
More information about how to configure Okteto Endpoints is available [on this link](core/ingress/automatic-ssl.mdx).

## Build your container images with Okteto Build

The Movies App is using pre-built images to deploy the Development Environment.
Let's change that, to make it possible to deploy the latest bits of your application.

To instruct Okteto to build your images, add the following content to your `okteto.yml` file:

```yaml
build:
  api:
    context: api
  frontend:
    context: frontend
```

And build your images with `okteto build`:

```console
$ okteto build
```

```console
 i  Using cindylopez @ okteto.sample.com as context
 i  Building 'api/Dockerfile' in tcp://buildkit.okteto.sample.com:443...
 ...
 ✓  Image 'registry.okteto.sample.com/cindylopez/movies-with-helm-api:okteto' successfully pushed
 i  Building 'frontend/Dockerfile' in tcp://buildkit.okteto.sample.com:443...
 ...
 ✓  Image 'registry.okteto.sample.com/cindylopez/movies-with-helm-frontend:okteto' successfully pushed
```

Finally, update your `deploy` section in the `okteto.yml` to use these images in the helm installation:

```yaml
deploy:
  - helm upgrade --install movies chart
    --set api.image=${OKTETO_BUILD_API_IMAGE}
    --set frontend.image=${OKTETO_BUILD_FRONTEND_IMAGE}
```

And run `okteto deploy` to upgrade your applications with the images built by Okteto.

## Configure an Okteto Variable for a database password

The file `chart/values.yaml` has a password hardcoded. This is not a good security practice :-)

Luckily, [Okteto Variables](admin/dashboard.mdx/#variables) can be used to inject sensitive information at runtime in your deploy commands.

Create an Okteto Variable called `MONGODB_PASSWORD` from the [Okteto Admin Dashboard](admin/dashboard.mdx/#variables), and use it this way in your `okteto.yml` file to configure the MongoDB password:

```yaml
deploy:
  - helm upgrade --install movies chart
    --set api.image=${OKTETO_BUILD_API_IMAGE}
    --set frontend.image=${OKTETO_BUILD_FRONTEND_IMAGE}
    --set mongodb.password=${MONGODB_PASSWORD}
```
