This guide explains how to deploy your first app to Okteto.

## Requirements

Before you start this tutorial, make sure you fulfill the following requirements:

- You have access to an Okteto instance. Otherwise, follow our [installation guide](get-started/overview.mdx)
- You have [configure the Okteto CLI](get-started/install-okteto-cli.mdx) with your Okteto instance

## What you will be building

This tutorial will guide you on how to deploy the Movies app to Okteto. The Movies app consists of:

- a React frontend
- a Node.js backend API
- a MongoDB database

In order to deploy the Movies app, your will create your first [Okteto Manifest](manifest/overview-manifest.mdx).
This is how your Okteto Manifest will look like at the end of the tutorial:

```yaml
build:
  api:
    context: api
  frontend:
    context: frontend

dependencies:
  mongodb:
    repository: https://github.com/okteto/mongodb
    wait: true

deploy:
  - name: Helm Install
    command: |
      helm upgrade --install movies chart \
        --set mongodb.password=${MONGODB_PASSWORD} \
        --set seed.enabled=true \
        --set api.image=${OKTETO_BUILD_API_IMAGE} \
        --set frontend.image=${OKTETO_BUILD_FRONTEND_IMAGE}

dev:
  api:
    command: yarn start
    sync:
      - api:/src
    forward:
      - 9229:9229

  frontend:
    image: okteto/node:14
    command: bash
    sync:
      - frontend:/usr/src/app
```

The meaning of the different sections is:

- `build`: Configure a list of images to build for your application
- `dependencies`: Configure a list of git repositories to deploy as part of your application
- `deploy`: Configure a list of commands to deploy your application
- `dev`: Configures hot realoding and debuggers for developing your application

Now you know what your are building, [start the tutorial](deploy.mdx) and deploy your app to Okteto!
