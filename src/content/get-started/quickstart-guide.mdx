## Installation Requirements

In order to fully install Okteto in your Kubernetes cluster, you will need the following:

- An Okteto License
- A Kubernetes cluster
- A working installation of kubectl
- A working installation of Helm v3 (v3.8 or higher)

#### Getting your Okteto License

A license is mandatory to use Okteto.

If you are interested in evaluating Okteto, [sign up for our free trial](https://www.okteto.com/free-trial/). No credit card required.

#### Kubernetes cluster

You need access to a Kubernetes cluster. Okteto supports Kubernetes versions 1.25 through 1.28.

#### Installing kubectl

Follow the official documentation for [installing kubectl](https://kubernetes.io/docs/tasks/tools/install-kubectl/). Once installed, configure `kubectl` to [talk to your Kubernetes cluster](https://cloud.google.com/kubernetes-engine/docs/how-to/cluster-access-for-kubectl#generate_kubeconfig_entry).

#### Installing Helm v3 (v3.8 or higher)

Follow the official documentation for [installing the latest release of Helm v3 (v3.8 or higher)](https://helm.sh/docs/intro/install/).


## Deploying Okteto

#### Configuration file

In order to install Okteto you need to first create a `config.yaml` file for the installation process.

Start by adding your license to this configuration file:

```yaml
license: <<your-okteto-license>>
```

Where `<<your-okteto-license>>` is the license key you have received by email when starting your trial.

#### Installing with Helm

Run the following `helm` commands to deploy Okteto in the `okteto` namespace:

```console
$ helm repo add okteto https://charts.okteto.com
$ helm repo update
$ helm install okteto okteto/okteto -f config.yaml --namespace=okteto --create-namespace
```

```console
Congratulations! Okteto is successfully installed!

Follow these steps to access your Okteto instance:

1. Start a port-forward to the ingress service by running:

   $ sudo kubectl port-forward service/okteto-ingress-nginx-controller 443:443 --namespace okteto

2. Access your Okteto instance at https://okteto.localtest.me/login#token=******

Follow these steps to configure the Okteto CLI:

1. Install the Okteto CLI:

   $ curl https://get.okteto.com -sSfL | sh

2. Configure the Okteto CLI context:

   $ okteto context use https://okteto.localtest.me --token ****** --insecure-skip-tls-verify

Happy coding!
```

Once the chart is installed, follow the onscreen instructions to access your Okteto instance using a port forward and a random token, install the Okteto CLI and configure the Okteto CLI to talk to your Okteto instance.

> **Important**: The default installation uses port-forwards and self-signed certificates to access your Okteto instance. It's designed for initial evaluations, but not for production usage. We highly advise following our [cloud provider guides](get-started/cloud-provider-guides/index.mdx) to complete your Okteto installation after your initial evaluation

You can validate your completed these steps properly by running:

```console
$ okteto context list
```

```console
https://okteto.localtest.me *    okteto-admin    tcp://buildkit.localtest.me:443    registry.localtest.me
```
