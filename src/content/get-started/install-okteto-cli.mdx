The Okteto CLI is our [open-source tool](https://github.com/okteto/okteto) that helps you develop your apps on Okteto.

This guide will teach how to get started with the Okteto CLI.

## Installing the Okteto CLI

Install the Okteto CLI following these steps:

### MacOS / Linux

```console
$ curl https://get.okteto.com -sSfL | sh
```

If you need a specific version you can set the OKTETO_VERSION environment variable:

```console
$ curl https://get.okteto.com -sSfL | OKTETO_VERSION=2.3.3 sh
```

You can also install it via [brew](https://brew.sh/) by running:

```console
$ brew install okteto
```

### Windows

Download https://downloads.okteto.com/cli/stable/okteto.exe and add it to your `$PATH`.

You can also install it via [scoop](https://scoop.sh/) by running:

```console
$ scoop install okteto
```

> For updating okteto with scoop, you might need to `scoop unhold okteto & scoop update okteto` or `scoop uninstall okteto & scoop install okteto`

### GitHub

Alternatively, you can directly download the binary [from GitHub](https://github.com/okteto/okteto/releases) or build it directly from the source code.


## Configuring the Okteto CLI

The first thing you need to do before using the Okteto CLI is to set the Okteto CLI context with your Okteto instance.
To do this, run the command below using your Okteto instance URL:

```console
$ okteto context use <<your-okteto-instance-url>>
```

> Follow our [quickstart installation guide](get-started/quickstart-guide.mdx) if you don't have access to an Okteto instance yet


You can confirm that your CLI is configured by running the command below:

```console
$ okteto context list
```
```console
Name                          Namespace       Builder                                 Registry
https://okteto.example.com *  cindylopez      tcp://buildkit.okteto.example.com:1234  registry.okteto.example.com
```

## Define Your Okteto Manifest

The Okteto CLI needs an [Okteto manifest](manifest/overview-manifest.mdx) to understand how to build, deploy and develop your application.
The Okteto manifest is required in all cases except when a Docker compose file exists.
If you don't have an Okteto manifest or a Docker Compose file in your git repository, we recommend to follow [this step by step tutorial](deploy-your-app/index.mdx) to deploy your app to Okteto.
