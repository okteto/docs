Okteto lets you develop applications against a remote Kubernetes cluster that your platform team manages. You write code locally, and Okteto syncs it into a Development Container running in the cluster, so your changes run in a production-like environment as you save them.

This section covers the developer workflow in order:

- [Quickstart](get-started/dev-quickstart.mdx) walks you through signing in, deploying a Development Environment from the Okteto Catalog, and running `okteto up` for the first time.
- [Using the Okteto CLI and Dashboard](get-started/using-okteto-cli-and-dashboard.mdx) explains the Okteto Manifest, the essential CLI commands, and how to manage Namespaces and environments from the Dashboard.
- [Advanced commands and concepts](get-started/advanced-commands-and-concepts.mdx) covers additional CLI commands and flags, the concepts behind Development Environments, and answers to common questions.

If you haven't installed the command-line tool yet, start by [installing the Okteto CLI](get-started/install-okteto-cli.mdx).
